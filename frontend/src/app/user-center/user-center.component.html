Multi-Agent Learning Platform

平台介绍

本平台是一个面向 多智能体（Multi-Agent） 学习与实验的交互式平台，帮助你直观理解多个 AI Agent 如何协作完成复杂任务。核心特性包括：

  • 可视化工作流编辑器 -- 拖拽式设计 Agent 协作流程图
  • 2D 工作流可视化 -- 实时观察 Agent 的运行状态与交互过程
  • 模版快速起步 -- 内置软件开发、新闻编辑、医疗诊断等场景模版
  • 条件分支与循环 -- 支持判断节点，实现动态工作流
  • 逐步调试 -- 单步执行、暂停、继续，深入理解每一步
  • 3D 躲猫猫实验 -- 在 3D 场景中观察多 Agent 的躲藏与搜索策略，研究信息共享对协作效率的影响

快速开始

方式一：从模版新建（推荐新手）

  1. 点击侧边栏 工作流 展开子菜单
  2. 点击 从模版新建
  3. 选择一个场景模版（如"软件开发场景"）
  4. 自动加载到编辑器，可直接运行

方式二：从零构建

  1. 点击侧边栏 工作流 展开子菜单
  2. 点击 新建工作流 进入编辑器
  3. 在工具栏输入 Agent 的 ID、Role、Prompt，点击"添加"
  4. 右键节点，添加边或条件边，连接各 Agent
  5. 设置任务目标，点击 进入场景 运行

3D 躲猫猫实验

3D 躲猫猫是一个多 Agent 策略实验场景，用于观察和研究躲藏者之间的信息共享对搜索效率的影响。

场景说明

  • 场景中有一个搜索者和多个躲藏者
  • 搜索者拥有 30 度视野范围，视线会被障碍物遮挡
  • 搜索者具有记忆能力，会记录已搜索过的位置
  • 躲藏者可以感知一定范围内（8米）的搜索者
  • 躲藏者有有限的移动次数（每局 2 次），可在搜索过程中选择移动或原地不动
  • 已被其他躲藏者占据的位置不能再被选择

游戏配置

选项 说明
躲藏者数量 可选择 1-5 名躲藏者，默认 3 名
信息共享 开启后，躲藏者之间可以共享"搜索者在我附近"的警告信息，包含发送者的位置和搜索者距离；关闭时躲藏者只能依靠自身感知

实验目的

通过对比"信息共享"开启与关闭时的搜索轮次，观察信息共享对躲藏者存活率的影响：

  • 开启信息共享：躲藏者可以收到队友的警告，更可能及时逃离搜索者
  • 关闭信息共享：躲藏者只能依靠自身感知，可能无法及时察觉搜索者靠近

学习建议

  1. 先跑模版：从"软件开发场景"开始，用逐步执行观察每个 Agent 的输入输出
  2. 修改 Prompt：尝试修改某个 Agent 的 Prompt，观察输出变化
  3. 设计自己的流程：从零构建一个工作流，理解条件分支和循环的作用
  4. 对比不同场景：观察软件开发、新闻编辑、医疗诊断中 Agent 协作模式的异同
  5. 信息共享实验：在 3D 躲猫猫中，分别开启和关闭信息共享，对比搜索者找到所有躲藏者所需的轮次

常见问题

Q: 如何保存我的工作流？

A: 点击编辑器工具栏的"保存"按钮保存到服务器，或点击"导出 JSON"下载到本地。下次可通过"加载"按钮或"我的工作流"恢复。

Q: 为什么 Agent 的回复不太准确？

A: Prompt 的质量直接影响输出。建议：1. 明确角色和输出格式；2. 提供具体示例；3. 避免模糊描述。系统已优化为传递完整上下文历史。

Q: 条件分支是怎么工作的？

A: 判断节点（Judge Agent）会根据前序输出生成一个 JSON 条件结果（如 {{ '{' }}"condition": "pass"{{ '}' }}），系统据此决定走哪条分支。你可以在条件边中配置每个条件值对应的目标节点。

Q: 3D 躲猫猫中信息共享具体共享了什么？

A: 开启信息共享后，当某个躲藏者感知到搜索者在其 8 米范围内时，其他躲藏者会收到警告信息，包含发送者的位置和搜索者距其的距离。例如："躲藏者2(位置:-5,8)报告: 搜索者距其3.5米"。收到警告的躲藏者可以据此决定是否移动到更安全的躲避点。

Q: 躲藏者能移动到其他躲藏者所在的位置吗？

A: 不能。每个躲避点同一时间只能容纳一名躲藏者。已被占据的位置不会出现在其他躲藏者的可选项中。
